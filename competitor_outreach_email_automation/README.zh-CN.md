# 竞品媒介线索与邮件外联自动化用例

本目录介绍一个“竞品公开线索发现 + 联系人富集 + 邮件外联 + 回信监听”的自动化用例。它适合品牌团队持续追踪竞品在媒体、榜单、评测、Deal 站和红人内容中的曝光，并把可合作的媒体机会转成可审核、可发送、可回溯的邮件队列。

## 适用场景

品牌团队希望从公开渠道发现竞品正在获得哪些媒体曝光，例如：

- 新闻报道
- Deal / Coupon 页面
- 礼品榜单或测评榜单
- 博客评测
- YouTube / Instagram / TikTok 红人内容
- DTC 或 Amazon 相关导购内容

人工做这件事通常会卡在三个地方：

1. 线索太分散，容易漏掉高价值页面。
2. 找联系人和判断是否值得联系很耗时。
3. 邮件发送、回信跟进和内部状态同步容易断档。

该自动化把这些步骤拆成可控流水线：先发现和评分，再生成待审邮件，最后在明确授权后才真实发送。

## 核心流程

```text
公开网页发现
  -> 竞品/媒体线索去重
  -> 机会评分与优先级排序
  -> 联系人邮箱富集
  -> 人工审核是否允许触达
  -> 生成邮件草稿或待发队列
  -> 明确授权后 SMTP 发信
  -> IMAP 监听回信
  -> 自动分类回复意图
  -> 生成回复草稿 / ClickUp 人工任务 / Feishu 协作看板
```

## 关键模块

### 1. 公开线索发现

从公开搜索、新闻源、媒体站点和已知竞品关键词中收集 URL。系统优先使用公开网页，不依赖社媒私域数据。

典型字段：

- 竞品品牌
- 来源 URL
- 来源类型：`news` / `ranking` / `deal` / `review` / `influencer`
- 标题与发布时间
- 市场与语言
- 是否可联系

### 2. 机会评分

系统根据品牌匹配度、媒体相关性、内容类型、近期程度、联系方式质量等维度打分。

常见状态：

- `WAITING_REVIEW`：需要人工确认是否允许触达
- `OUTREACH_READY`：已允许触达，可生成首封邮件
- `OUTREACH_SENT`：已发送
- `WAITING_MEDIA_REVIEW`：媒体内部评估中
- `QUOTE_DISCUSSION`：报价沟通中
- `SAMPLE_SHIPMENT`：样品寄送跟进

### 3. 联系人富集

自动从公开网页、作者页、媒体联系页等位置提取可联系邮箱，并按质量分层：

- `DIRECT_EMAIL`：个人或编辑邮箱，优先使用
- `DOMAIN_EMAIL`：站点通用域名邮箱，可作为次选
- `CONTACT_FORM`：只有表单，需人工判断
- `SOCIAL_ONLY`：仅社媒联系方式，不自动发信
- `NO_CONTACT`：无可用联系方式

### 4. 邮件生成与发送

默认只生成待发邮件，不直接发送。真实发信必须切换到 live 模式，并要求操作者明确确认。

邮件类型：

- 首封建联邮件
- Follow-up 邮件
- 报价信息补充请求
- 样品寄送确认回复
- 不匹配/婉拒回复

安全边界：

- 默认 `prepare` 模式：只生成队列，不发信。
- `live` 模式才调用 SMTP。
- 只给审核通过、联系方式有效、未退信、未在黑名单中的对象发信。
- 公共角色邮箱、退信邮箱和低质量联系人默认不进入自动发送。

### 5. 回信监听与后续动作

系统通过 IMAP 读取回信，识别回复意图并生成下一步动作：

- 询问报价：生成报价补充请求草稿
- 提供报价：转人工判断预算
- 需要样品：创建样品寄送人工任务
- 会议邀约：转人工确认时间
- 自动回复/OOO：只记录，不推进
- 拒绝合作：标记为已关闭

## 推荐运行模式

### 日常安全模式

```text
mail_mode = prepare
clickup_mode = prepare
sync_feishu = true
```

适合每天运行，用于生成待审队列、邮件草稿、回复摘要和状态看板。

### 明确授权后的发信模式

```text
mail_mode = live
max_send_per_run = 8
skip_bounced_contacts = true
require_approved_status = true
```

适合人工确认本轮名单后执行。建议限制单次发送数量，避免误发或过度触达。

## 输出看板

建议同步到 Feishu 或内部看板的表格：

- 线索总表：公开链接、竞品、来源、评分、状态
- 邮件队列：收件人、主题、邮件类型、发送状态
- 回信分层：回复时间、回复人、分类、下一步动作
- 每日状态：发信数、退信数、新回信数、同步状态、错误摘要
- 人工任务：报价、会议、样品寄送、不可自动判断事项

## 人工介入点

这套自动化不是“全自动群发器”。建议保留以下人工节点：

- 首次触达前确认 `approval_status=APPROVED`
- 报价与预算相关事项人工确认
- 样品寄送信息人工确认
- 会议邀约人工确认
- 媒体资质或联系人质量不清楚时人工复核

## 不建议自动化的情况

- 未确认来源合法性的联系人
- 明显通用邮箱或客服邮箱
- 已退信或投诉风险较高的邮箱
- 对方已明确拒绝合作
- 需要承诺价格、样品库存、投放预算或合同条款的回复

## 接入前准备

1. 准备竞品关键词与媒体 watchlist。
2. 准备品牌介绍、产品卖点、媒体包链接和样品政策。
3. 配置 SMTP，但默认使用 prepare 模式测试。
4. 配置 IMAP，用于监听回信。
5. 建立 Feishu/ClickUp 看板字段映射。
6. 明确单次最大发送量和退信黑名单规则。

## 成功标准

一次健康运行至少应满足：

- 邮件队列可回溯，每封邮件都有来源线索。
- 未经审核的对象不会被真实发送。
- 退信和低质量邮箱被过滤。
- 回信可以被分类到报价、样品、会议、拒绝或自动回复。
- 即使 Feishu/ClickUp 同步失败，本地 CSV/日志仍保留完整状态。
